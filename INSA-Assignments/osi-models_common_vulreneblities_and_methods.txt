What is the OSI Model?

The OSI Model (Open Systems Interconnection model) is a 7-layer framework that explains how data moves across a network—from one device to another.

The 7 Layers of the OSI Model

1.Physical Layer (Layer 1)
-----------------------------------------------------------------------------------------------------------------------------------------------------
What it does: Transmits raw bits over physical media (cables, radio signals).

Common vulnerabilities:
----------------------
Cable tapping / interception
Hardware tampering
Signal jamming (wireless)

Attack methods:
--------------
Physically accessing network cables
Using radio frequency jammers
Installing rogue devices (e.g., USB implants)

Tools used:
-----------
Hardware sniffers
RF jammers
USB drop attacks

2. Data Link Layer (Layer 2)
-----------------------------------------------------------------------------------------------------------------------------------------------------
What it does: Handles MAC addresses and switching (local network communication).

Common vulnerabilities:
----------------------
MAC spoofing
ARP poisoning
VLAN hopping

Attack methods:
--------------
Pretending to be another device
Intercepting traffic via ARP spoofing
Jumping between VLANs

Tools used:
----------
Ettercap
dsniff
Yersinia

3. Network Layer (Layer 3)
----------------------------------------------------------------------------------------------------------------------------------------------------
What it does: Routing packets using IP addresses.

Common vulnerabilities:
----------------------
IP spoofing
Routing attacks (BGP hijacking)
ICMP abuse (ping floods)

Attack methods:
--------------
Forging IP packets
Redirecting traffic through malicious routes
Launching DDoS attacks

Tools used:
-----------
Scapy
hping3
LOIC

4. Transport Layer (Layer 4)
------------------------------------------------------------------------------------------------------------------------------------------------------
What it does: Ensures reliable data transfer (TCP/UDP).

Common vulnerabilities:
---------------------
SYN flood attacks
Session hijacking
Port scanning exposure

Attack methods:
--------------
Overwhelming servers with half-open connections
Taking over TCP sessions
Discovering open ports

Tools used:
----------
Nmap
Metasploit
Wireshark

5. Session Layer (Layer 5)
--------------------------
What it does: Manages sessions between applications.

Common vulnerabilities:
-----------------------
Session hijacking
Session fixation
Weak session tokens

Attack methods:
--------------
Stealing session IDs
Reusing authentication tokens
Predicting session identifiers

Tools used:
----------
Burp Suite
OWASP ZAP

6. Presentation Layer (Layer 6)

What it does: Handles data formatting, encryption, and encoding.

Common vulnerabilities:
----------------------
Weak encryption
SSL/TLS misconfigurations
Data serialization flaws

Attack methods:
--------------
Man-in-the-middle (MITM) attacks
Breaking weak encryption
Exploiting improper encoding

Tools used:
----------
SSLstrip
OpenSSL

7. Application Layer (Layer 7)
------------------------------
What it does: Interfaces directly with user applications (HTTP, FTP, DNS, etc.).

Common vulnerabilities:
----------------------
SQL injection
Cross-site scripting (XSS)
Remote code execution (RCE)
DNS poisoning

Attack methods:
---------------
Injecting malicious queries
Executing scripts in user browsers
Exploiting app logic flaws

Tools used:
----------
sqlmap
Nikto
DirBuster



conclusion
<<.......>>
Lower layers (1–3): Focus on hardware and routing → attacked via spoofing, interception, and disruption
Middle layers (4–5): Focus on connections/sessions → attacked via hijacking and flooding
Upper layers (6–7): Focus on data and applications → attacked via injections, encryption flaws, and logic bugs
